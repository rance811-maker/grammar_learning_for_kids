# Grammar Quest 账号系统设置说明

账号系统基于 **Supabase Auth**（邮箱 + 密码登录），密码找回邮件由 Supabase 原生发送。
学习数据存在 `gq_profiles` 表，靠行级安全(RLS)保证每个人只能读写自己的数据。
（表名用 `gq_profiles` 是为了避开项目里可能已存在的同名 `profiles` 表。）

## 一次性设置（在 Supabase 控制台操作）

### ① 建表
左侧 **SQL Editor → New query**，把 `docs/supabase-setup.sql` 整个粘贴进去，点 **Run**。
（可重复运行，安全。）

### ② 配置站点网址（找回密码邮件要用）
左侧 **Authentication → URL Configuration**：
- **Site URL** 填：`https://rance811-maker.github.io/grammar_learning_for_kids/`
- **Redirect URLs** 里也把上面这个网址加进去（点 Add URL）。

> 找回密码 / 确认邮件里的链接会跳回这个网址，所以必须先加进白名单。

### ③（建议）关闭邮箱确认，让小朋友注册后能立刻玩
左侧 **Authentication → Providers → Email**：
- 把 **Confirm email** 关掉（OFF）。

这样注册后直接进入，不用先去邮箱点确认。
（如果保持开启也能用，只是注册后要先去邮箱点一下确认链接才能登录。）

## 使用方式

- **访客模式**：打开就能玩，进度存在本机。
- **注册**：名字（昵称）+ 家长邮箱 + 密码（≥6 位）。注册时本机已有的进度会带进新账号。
- **登录**：邮箱 + 密码。登录后切换成账号里云端保存的进度，多设备同步。
- **忘记密码**：在登录框填入注册邮箱 → 点"忘记密码？" → 收邮件 → 点链接 → 设置新密码。

## 关于发信额度

Supabase 自带的邮件发送有较低的频率限制（适合小规模使用）。
如果以后用户变多、发信不够用，可在 **Authentication → Emails / SMTP Settings**
里配置自定义 SMTP（如 Resend、Gmail 等）来提高额度。

## 备注

- 旧版自建账号表 `kids` 和相关函数已不再使用，可在 `supabase-setup.sql` 末尾
  取消注释来删除它们（可选）。
- `js/supabaseConfig.js` 里是项目 URL 和公开 key（publishable key 本就可公开，
  真正的安全由 RLS 保证）。
